## 一、项目全景概览

### 1.1 项目定位

这是一个**美股异动实时监控+AI助手**系统，核心价值：

- **自动抓取**：从Stock Titan网站实时获取美股新闻
- **智能分析**：统计股票异动次数，识别重要信号
- **实时推送**：支持钉钉/微信/Telegram多渠道通知
- **AI助手**：基于MCP协议，让AI能查询股票数据

**职责**：为AI助手提供工具调用能力

**核心类**：

| 类名 | 职责 | 关键注解 |
|------|------|---------|
| `StockTool` | 股票查询工具 | `@Tool(description = "...")` |

| `DateTool` | 日期转换工具 | `@Tool(description = "...")` |

**什么是MCP？**

MCP（Model Context Protocol，模型上下文协议）是Anthropic推出的开放协议，让AI助手能调用外部工具。

**类比理解**：
- 就像浏览器通过HTTP协议访问网站
- AI通过MCP协议调用你的Java方法

#### 6.2.2 RSS核心服务

```java
@Service
public class RssServiceImpl implements RssService {

    public static final String RSS_URL = "https://www.stocktitan.net/rss";

    @Override
    public void displayRss() throws Exception {
        List<USStockMsg> stockMsgList = new ArrayList<>();

        // 1. 抓取RSS
        List<SyndEntry> rssList = this.fetchRssReed(RSS_URL);

        if (rssList == null || rssList.isEmpty()) {
            System.out.println("RSS 列表为空");
            return;
        }

        // 2. 遍历处理每条RSS
        for (SyndEntry entry : rssList) {
            USStockRss stockNews = new USStockRss();

            // 2.1 解析标题
            String title = entry.getTitle();
            String titleEn = getStockTitle(title);
            stockNews.setTitle(titleEn);

            // 2.2 解析链接
            stockNews.setLink(entry.getLink());

            // 2.3 解析时间（GMT和北京时间）
            Date gmtDateTemp = entry.getPublishedDate();
            LocalDateTime gmtDate = GMTDateConverter.convertGmt(gmtDateTemp);
            stockNews.setPubDateGmt(gmtDate);
            stockNews.setPubDateBj(GMTDateConverter.convertGmtToBeijing(gmtDateTemp));

            // 2.4 解析股票代码
            String stockCode = getStockCode(title);
            stockNews.setStockCode(stockCode);

            // 2.5 去重检查
            if (stockService.isStockNewsExist(stockCode, stockNews.getLink())) {
                continue;  // 已存在，跳过
            }

            // 2.6 爬取标签（带异常处理）
            try {
                List<String> tagsList = StockTitanCrawler.getTags(titleEn);
                stockNews.setTags(getTagsZh(tagsList));
            } catch (Exception e) {
                // 429反爬异常，忽略标签
                stockNews.setTags("");
            }

            // 2.7 保存到数据库
            stockService.saveStockNews(stockNews);

            // 2.8 构建消息对象
            USStockMsg stockMsg = new USStockMsg();
            BeanUtils.copyProperties(stockNews, stockMsg);

            // 2.9 统计异动次数
            Long counts24Hour = stockService.getStockUnusualCounts(
                stockNews,
                GMTDateConverter.minus24Hour(gmtDate),
                GMTDateConverter.plus1Minute(gmtDate)
            );
            stockMsg.setCounts24Hour(counts24Hour.intValue());

            // 3. 批量推送
            if (!stockMsgList.isEmpty()) {
                List<List<USStockMsg>> batchList = splitList(stockMsgList);
                for (List<USStockMsg> batch : batchList) {
                    dingTalkApi.sendTextMessage(
                        dingTalkApi.formatStockInfoFromList(batch)
                    );
                }
            }
        }
    }
}
```

#### 6.2.3 关键方法解析

**解析股票代码**：
```java
// 标题格式："Apple Hits New High | AAPL Stock News"
private String getStockCode(String title) {
    String[] titleArr = title.split("\\|");
    String stockStr = titleArr[titleArr.length - 1];  // "AAPL Stock News"
    String[] stockCodeArr = stockStr.split("Stock News");
    return stockCodeArr[0].trim();  // "AAPL"
}
```

**消息拆分（避免过长）**：
```java
public static <T> List<List<T>> splitList(List<T> input) {
    List<List<T>> result = new ArrayList<>();
    int size = input.size();

    if (size <= 10) {
        result.add(new ArrayList<>(input));
    } else {
        // 每10个拆分一组
        for (int i = 0; i < size; i += 10) {
            int end = Math.min(i + 10, size);
            result.add(new ArrayList<>(input.subList(i, end)));
        }
    }
    return result;
}
```

## 七、MCP AI模块解析

### 7.1 什么是MCP？

**MCP（Model Context Protocol）** 是一种开放协议，标准化了AI模型与外部工具的交互方式。

**类比**：
| 场景 | 协议 | 作用 |
|------|------|------|
| 浏览器访问网站 | HTTP | 获取网页内容 |
| AI调用工具 | MCP | 获取数据/执行操作 |

### 7.2 MCP工具实现

```java
@Component
@Slf4j
public class StockTool {

    @Resource
    private StockService stockService;

    @Tool(description = "根据股票代码查询股票信息数据")
    public List<USStockRss> queryStockInfoByCode(String stockCode) {
        log.info("========== 调用MCP工具：queryStockInfoByCode() ==========");
        return stockService.queryStock(stockCode);
    }

    @Tool(description = "根据股票代码查询某段日期内的股票信息数据")
    public List<USStockRss> queryStockInfoByCodeBetweenDate(
            String stockCode, String startDate, String endDate) {
        return stockService.queryStockBetweenDate(stockCode, startDate, endDate);
    }

    @Tool(description = "查询某日期段内的哪些股票出现的次数超过指定的目标次数")
    public List<StockCounts> queryStockCountsBetweenDate(
            Integer targetCounts, String startDate, String endDate) {
        return stockService.queryStockCountsBetweenDate(targetCounts, startDate, endDate);
    }

    @Tool(description = "根据标题关键字来查询股票信息数据")
    public List<USStockRss> queryStockByTitleKeywords(List<String> titleKeywords) {
        return stockService.queryStockByTitleKeywords(titleKeywords);
    }
}
```

            "https://api.telegram.org/bot%s/sendMessage?chat_id=%s&text=%s",
            botToken, chatId, URLEncoder.encode(message, "UTF-8")

    }
}

## 十、常见问题排查

### 10.1 推送问题

| 问题 | 排查步骤 |
|------|---------|
| 钉钉收不到消息 | 1. 检查token和secret<br>2. 确认加签方式正确<br>3. 查看钉钉群机器人日志 |
| Telegram收不到 | 1. 检查bot token<br>2. 确认chat_id正确<br>3. 检查网络是否能访问telegram |
| 微信不工作 | 1. 确认Windows系统<br>2. 检查微信版本<br>3. 查看Python脚本日志 |

### 10.2 RSS问题

| 问题 | 排查步骤 |
|------|---------|
| RSS解析失败 | 1. 检查网络连接<br>2. 确认RSS URL可访问<br>3. 查看Rome库异常信息 |
| 标签爬取失败 | 1. 检查是否触发429<br>2. 增加请求间隔<br>3. 更换User-Agent |

### 10.3 数据库问题

| 问题 | 排查步骤 |
|------|---------|
| 连接失败 | 1. 检查MySQL服务<br>2. 确认端口、用户名、密码<br>3. 查看防火墙设置 |
| 中文乱码 | 1. 确认utf8mb4编码<br>2. 检查连接字符串 |

### 10.4 MCP问题

| 问题 | 排查步骤 |
|------|---------|
| MCP连不上 | 1. 检查端口7070<br>2. 确认服务已启动<br>3. 查看MCP配置 |
| AI不调用工具 | 1. 检查@Tool描述是否清晰<br>2. 确认参数类型正确<br>3. 查看MCP日志 |
